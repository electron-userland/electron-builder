# Migrating from v26 to v27

v27 is a major release that migrates the entire electron-builder package ecosystem to native ES modules (ESM) and drops support for Node.js versions below 22.12. Most projects will only need to update their Node.js version; the actual build configuration is unchanged.

## Breaking changes at a glance

| Change | Action required |
|--------|----------------|
| Node.js >=22.12.0 required | Update your runtime and CI environment |
| All packages are now native ESM | Usually none — CJS `require()` still works on Node >=22.12 |
| `electronCompile` config option removed | Migrate to a modern bundler (see below) |
| `electron-forge-maker-*` are now ESM | Usually none — same API, same `export default` shape |

---

## 1. Update Node.js to >=22.12.0

v27 requires **Node.js 22.12.0 or later**. This is the specific version where Node's [`require(esm)`](https://nodejs.org/en/blog/release/v22.12.0) support was stabilized (no flags needed), which is how electron-builder packages remain usable from both ESM and CJS consumers.

Update your local environment:
```bash
# using nvm
nvm install 22
nvm use 22

# using fnm
fnm install 22
fnm use 22
```

Update your CI configuration (GitHub Actions example):
```yaml
- uses: actions/setup-node@v4
  with:
    node-version: '22'
```

Update your `package.json` if you declare an `engines` field:
```json
{
  "engines": {
    "node": ">=22.12.0"
  }
}
```

---

## 2. ESM output — what changes for your project

electron-builder packages now ship as native ES modules. On **Node >=22.12.0**, both import styles continue to work with no changes to your code:

**CJS `require()` — still works**
```js
// CommonJS — works unchanged on Node >=22.12
const { build, Platform } = require("electron-builder")
```

**ESM `import` — now the preferred style**
```js
// ES module
import { build, Platform } from "electron-builder"
```

**TypeScript programmatic usage**
```ts
import { build, Configuration, Platform } from "electron-builder"

const config: Configuration = {
  appId: "com.example.myapp",
  mac: { target: "dmg" },
}

await build({ targets: Platform.MAC.createTarget(), config })
```

If your project already uses `"type": "module"` or top-level `await`, no further changes are needed. If it uses CJS and you're on Node >=22.12, `require()` keeps working.

---

## 3. Remove `electronCompile` usage

The `electronCompile` configuration option has been removed. `electron-compile` is an unmaintained library that has not had a release since 2019.

**Remove from your configuration:**
```json5
{
  "build": {
    "electronCompile": true  // ← remove this line
  }
}
```

If your project currently relies on `electron-compile` for source compilation, migrate to a modern build tool. Recommended options:

- **[Vite](https://vitejs.dev/)** — fast, first-class ESM support, excellent Electron integration via [`electron-vite`](https://electron-vite.org/)
- **[esbuild](https://esbuild.github.io/)** — extremely fast, simple configuration
- **[webpack](https://webpack.js.org/)** — widely used, mature ecosystem, [`electron-webpack`](https://webpack.electron.build/) for Electron-specific setup

A minimal Vite + Electron setup:
```bash
npm create electron-vite@latest my-app
```

---

## 4. `electron-forge-maker-*` plugins

The four forge-maker plugins (`electron-forge-maker-appimage`, `electron-forge-maker-nsis`, `electron-forge-maker-nsis-web`, `electron-forge-maker-snap`) are now native ES modules. The public API is identical:

```js
// ESM (new preferred style — same shape as before)
import maker from "electron-forge-maker-nsis"
```

If your Electron Forge configuration references these makers, no change is needed — Forge loads them via dynamic `import()` internally.

---

## 5. Programmatic API — no changes

The `build()` function signature, all configuration options, and all exported types are unchanged. Only the import mechanism differs:

```js
// Before (v26, CJS)
const { build } = require("electron-builder")

// After (v27) — CJS still works on Node >=22.12
const { build } = require("electron-builder")

// After (v27) — ESM (preferred)
import { build } from "electron-builder"
```

## 6. CI environments

If you use Docker or CI images, confirm the Node.js version is 22.12 or later.

**GitHub Actions**
```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/setup-node@v4
        with:
          node-version: '22'
```

**Docker**
```dockerfile
FROM node:22-bookworm-slim
```

electron-builder's own Docker images for Linux builds have been updated to Node 22.

- [ ] Node.js runtime updated to >=22.12.0
- [ ] CI/CD `node-version` updated to `22`
- [ ] `electronCompile: true` removed from build config (if present)
- [ ] Migrated off `electron-compile` to a modern bundler (if applicable)
- [ ] `"engines"` field in your `package.json` updated (if declared)
